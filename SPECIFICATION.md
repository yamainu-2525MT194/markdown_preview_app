## 1. プロジェクト概要

| 項目 | 内容 |
|------|------|
| アプリ名 | Markdown Editor |
| 種別 | ローカル動作デスクトップアプリケーション |
| 対応OS | Windows / Linux |
| アーキテクチャ | フロントエンド・バックエンド分離型（Tauri IPC）|

処理速度に優れたローカル動作のマークダウンエディタ。将来的なクラウド（Web）展開を見据え、React フロントエンドと Rust バックエンドを IPC で明確に分離した設計を採用。

### 2.2 機能一覧

#### ヘッダー

| No. | 機能 | 詳細 |
|-----|------|------|
| H-1 | ファイル名表示 | 開いているファイル名を表示 |
| H-2 | 未保存インジケータ | 変更がある場合、ファイル名の横に黄色ドットを表示 |
| H-3 | フォルダを開く | ダイアログでローカルディレクトリを選択し、ファイルツリーに表示 |
| H-4 | 新規作成 | 新規ファイルの保存先をダイアログで指定して作成・開く |
| H-5 | 保存 | 現在のファイルを上書き保存（未変更時は非活性） |
| H-6 | PDF 出力 | プレビューペインの内容を OS の印刷ダイアログ経由で PDF 保存 |

#### 左ペイン（ファイルツリー）

| No. | 機能 | 詳細 |
|-----|------|------|
| L-1 | ディレクトリ表示 | 選択したフォルダ配下のファイル・フォルダをツリー表示（最大3階層） |
| L-2 | フォルダ展開 | クリックでフォルダの展開/折りたたみ |
| L-3 | ファイルを開く | .md ファイルをクリックすると中央エディタに内容を読み込む |
| L-4 | ソート | フォルダ優先・名前昇順で自動ソート |
| L-5 | 隠しファイル除外 | `.` で始まるファイル・フォルダは非表示 |
| L-6 | アクティブ強調 | 現在開いているファイルをハイライト表示 |

#### 中央ペイン（エディタ）

| No. | 機能 | 詳細 |
|-----|------|------|
| C-1 | マークダウン入力 | テキストエリアへの自由入力 |
| C-2 | タブ補完 | Tab キーで 2 スペースを挿入（フォーカス移動しない） |
| C-3 | モノスペースフォント | JetBrains Mono / Fira Code / Consolas 等を使用 |
| C-4 | キーボードショートカット | **Ctrl+S** でファイル保存 |
| C-5 | テキスト装飾ボタン | 選択したテキストに対して、マークダウン記法を自動付与するボタン群 |
| C-6 | 見出し（H1〜H3） | 選択行の先頭に `#`, `##`, `###` を挿入、またはトグルする |
| C-7 | 太字 / 斜体 | 選択テキストを `**テキスト**`（太字）、`*テキスト*`（斜体）で囲む |
| C-8 | リスト（箇条・番号） | 選択行の先頭に `-` または `1.` を挿入する |
| C-9 | 引用 / コードブロック | 選択行を `>` で引用化、または ` ``` ` でコードブロック化する |
| C-10| リンク / 画像 | `[リンク](URL)` または `![画像](画像URL)` の雛形を挿入する |

#### 右ペイン（プレビュー）

| No. | 機能 | 詳細 |
|-----|------|------|
| P-1 | リアルタイムプレビュー | 入力と同時にマークダウンを HTML レンダリング |
| P-2 | GFM 対応 | テーブル・チェックボックス・取り消し線などの GitHub Flavored Markdown |
| P-3 | シンタックスハイライト | コードブロックを highlight.js でハイライト（github-dark テーマ） |
| P-4 | PDF 印刷スタイル | 印刷時はプレビュー内容のみを白背景で出力 |

### 2.3 IPC コマンド（Rust バックエンド）

| コマンド | 引数 | 戻り値 | 役割 |
|---------|------|--------|------|
| `read_dir` | `path: &str` | `Vec<FileInfo>` | ディレクトリを再帰的に読み込み、ファイル一覧を返す |
| `read_file` | `path: &str` | `String` | テキストファイルの内容を読み込む |
| `save_file` | `path: &str, content: &str` | `()` | ファイルを上書き保存（親ディレクトリも自動生成） |

```rust
// FileInfo 構造体（JSON シリアライズ）
struct FileInfo {
    name: String,
    path: String,
    is_dir: bool,
    children: Option<Vec<FileInfo>>,
}
```

### Rust インストール（Windows）

```powershell
# rustup-init.exe をダウンロードして実行
Invoke-WebRequest -Uri "https://static.rust-lang.org/rustup/dist/x86_64-pc-windows-msvc/rustup-init.exe" -OutFile "$env:TEMP\rustup-init.exe"
& "$env:TEMP\rustup-init.exe" -y --default-toolchain stable
```

### 開発コマンド

```bash
# 依存関係のインストール（初回）
npm install

# 開発サーバー起動（ホットリロード付き）
npm run tauri dev

# プロダクションビルド
npm run tauri build

# フロントエンドのみビルド確認
npx vite build

# Rust バックエンドのコンパイル確認のみ
cd src-tauri && cargo check
```

## 6. PDF エクスポート方式

外部バイナリ（Chrome ヘッドレス・wkhtmltopdf 等）への依存を排除するため、ブラウザ標準の `window.print()` を採用。

```
① ユーザーが「PDF」ボタンをクリック
      ↓
② window.print() を呼び出し
      ↓
③ CSS @media print により
   プレビューペインのみを白背景で表示
      ↓
④ OS の印刷ダイアログが開く
      ↓
⑤ 印刷先で「PDF に保存」を選択して出力
```

**メリット**: 追加依存ゼロ・クロスプラットフォーム対応・CSS で印刷スタイルを完全制御可能
